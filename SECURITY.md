# Security Policy

## Supported Use
This project is intended for local, personal use on your own machine.
Because it may involve local browser sessions, webhooks, and runtime data, please handle it carefully.

## What to Keep Private
Never publish or share:
- browser session data
- cookies or exported session snapshots
- webhook URLs
- local runtime databases
- logs containing personal or account information
- screenshots that expose account details

## Reporting a Vulnerability
If you believe you found a security issue, please avoid filing a public GitHub issue with sensitive details.

Instead, report it privately with:
- a short summary of the issue
- affected files or flows
- reproduction steps
- impact assessment
- any suggested mitigation

If you do not yet have a dedicated security contact channel, use a private GitHub contact path available to the repository owner, or establish one before encouraging public contributions.

## Out-of-Scope Reports
The following are generally out of scope unless they cause a concrete security impact:
- target website layout changes
- session expiration during normal use
- anti-automation defenses on third-party sites
- failures caused by invalid local configuration

## Safe Disclosure
Please give the maintainer reasonable time to review and address the issue before public disclosure.
